ADB para desarrolladores en React Native

17 enero 2020
react-native
consola

En el tiempo que llevo desarrollando aplicaciones mviles con React Native muchas veces he tenido que abrir AndroidStudio para poder hacer algo relacionado con el dispositivo (emulador o dispositivo fsico).

Para aquellos que alguna vez han desarrollado para Android saben que AndroidStudio tarda un par de minutos en iniciar y que consume mucha memoria. Tiempo y recursos que no valen la pena ser utilizados para tareas sencillas y repetitivas...

Aprender un par de comandos de ADB me ha ayudado a hacer dichas tareas de manera ms sencilla y rpida usando la lnea de comandos, sin mencionar que me gusta mucho hacer cosas con la terminal :P.

Qu es ADB?

Segn la definicin oficial, Android Debug Bridge es una herramienta de lnea de comandos que permite interactuar con el dispositivo con el que ests debuggeando una aplicacin Android en desarrollo. Bsicamente, un comando.

ADB tiene muchas opciones y argumentos que permiten ejecutar tareas diferentes en el emulador o dispositivo fsico. Pero la estructura bsica es:

adb [opcin] comando

El argumento opcin es innecesario si solo tienes un dispositivo o emulador conectado y comando es la accin que quieres ejecutar (es un argumento requerido).

Hay una lista muy larga de comandos que puedes ejecutar con adb para interactuar con el almacenamiento del dispositivo, para configurar los puertos, configurar el WiFi o las opciones de red y hasta tomar screenshots o grabar la pantalla del dispositivo.

Para ver todas las alternativas puedes ejecutar desde una lnea de comandos

adb --help

o consultar la documentacin oficial. En este post voy a describir solo unas cuantas que uso frecuentemente...

1. Port Forwarding

Este es un clsico, configurar la redireccin de puertos. Si usas librerias como Storybook o Reactotron te encuentras con que necesitas comunicar el puerto del dispositivo que usan estas librerias a un puerto de tu maquina de desarrollo, adb te ayuda con esto!

adb reverse <puerto-remoto> <puerto-local>

En el caso de Storybook el puerto remoto es tcp:7007, por lo que el comando adb es:

adb reverse tcp:7007 tcp:7007

2. Abrir el men de desarrollo

Si, ya s que esto mismo se puede hacer con la combinacin de teclado Ctrl + M (o Cmd + M para los usuarions de macOS). Pero yo lo hago con lnea de comandos porque... se puede! xD

adb shell input keyevent 82

3. Copiar archivos desde y hacia el almacenamiento del dispositivo

Para copiar un archivo hacia el almacenamiento del dipositivo:

adb push <archivo-local> <archivo-dispositivo>

Por ejemplo:

adb push ~/Downloads/image.png /sdcard/Downloads/image.png

Para copiar un archivo desde el almacenamiento del dispositivo hacia tu mquina de desarrollo:

adb pull <archivo-disposivo> <archivo-local>

Por ejemplo:

adb pull /sdcard/Downloads/voice.aac ~/Downloads/voice.aac

4. Instalar una app

Si tienes un instalador .apk vlido que quisieras instalar en un dispositivo desde la lnea de comandos:

adb install <archivo-apk>

5. Sacar screenshot

Es til tambin tomar una captura de pantalla del dispositivo en un determinado momento para documentar el estado de la aplicacin. Se logra facilmente con la siguiente sintaxis del comando:

adb shell screencap <nombre-de-archivo>

Por ejemplo, el resultado de:

adb shell screencap /sdcard/Pictures/captura-pantalla.png

es una imagen en /sdcard/Pictures con el nombre captura-pantalla.png.

6. Grabar video de la pantalla

As como se puede obtener una captura de pantalla, es tambin posible grabar un video de la pantalla.

adb shell screenrecord <nombre-de-archivo>

Para parar la grabacin basta con presionar la combinacin de teclado Ctrl + C.

Por ejemplo:

adb shell screenrecord /sdcard/Video/record.mp4


# 10 segundos despus
Ctrl + C

Bonus!

Este no es un comando de ADB en s, pero es posible abrir un emulador Android mediante la lnea de comandos.

En el directorio donde est instalado el Android SDK y dentro del directorio emulator (/path/to/Android/Sdk/emulator) existe un ejecutable llamado emulator con el cual se puede levantar un emulador:

cd /path/to/Android/Sdk/emulator
./emulator @<nombre-del-emulador>

Para tener una lista de los emuladores disponibles para ejecutar:

./emulator -list-avds

En mi caso la salida de este ltimo comando es:

# Example Response!
Pixel2
Pixel2old
Pixel_2_API_25
Pixel_2_API_26

Por lo que un ejemplo de comando para ejecutar el emulador Pixel2 es:

emulator @Pixel2

Por facilidad yo cre un link simbolico que apunta al ejecutable /path/to/Android/Sdk/emulator/emulator ;)

Bsicamente esos son los comandos que uso diaramente para desarrollar con React Native para Android. Saludos!
